Layer 1 & 2

Native Account Adapters

We bypass Bot APIs entirely. We use the protocols humans use.

  • Telegram: Full MTProto via gotd/td. We handle DC migrations and session rotation natively.
  • WhatsApp: Reverse-engineered whatsmeow. We act as a linked device.
  • Matrix: Production-grade mautrix-go.

Isolated Supervisors: Every adapter runs in its own goroutine with exponential backoff. If Telegram crashes, Matrix stays alive. No shared failures.

Layer 3

Universal Language

Every proprietary event is ruthlessly squashed into a single, versioned JSON schema.

ULIDs: We use time-sortable IDs. Database pagination is O(log n) without secondary indexes.

Layer 4

The Sovereign Store

// SQLite WAL Mode
PRAGMA journal_mode=WAL;
// One writer, many readers
SetMaxOpenConns(1);

GOAL: 100% data integrity under concurrent load.

Alice's messages live in a local SQLite DB powered by modernc.org/sqlite (Pure Go).

The Solution: A single StoreWriter Goroutine. Adapters push to a buffered channel; one goroutine flushes them in batches.

Zero lag. Zero database locks.
